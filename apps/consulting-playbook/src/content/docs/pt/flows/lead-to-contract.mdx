## Estados do Lead

```
new → nurturing → qualified → [won | lost | disqualified]
         ↑
    (requalifica após nurture)
```

## Sequência de Nutrição — Configuração no Directus

### `Lead_Nurture_Sequences`
| Campo | Tipo | Exemplo |
|---|---|---|
| `name` | string | "Waitlist Business PT" |
| `vertical` | enum | `business` / `church` / `media` |
| `trigger` | enum | `waitlist_signup` / `disqualified` |

### `Lead_Nurture_Steps`
| Campo | Tipo | Descrição |
|---|---|---|
| `step_order` | int | Ordem de envio |
| `delay_days` | int | Dias após o step anterior |
| `channel` | enum | `email` / `whatsapp` |
| `subject_template` | string | Assunto com variáveis |
| `body_template` | string | Corpo com `{name}`, `{company}`, `{vertical}` |

## Qualificação BANT (SalesQualifier)

| Dimensão | Pergunta padrão | Qualificador |
|---|---|---|
| **Budget** | "Qual o orçamento disponível para o projeto?" | > R$3k |
| **Authority** | "Você é o decisor ou precisa de aprovação?" | Decisor direto |
| **Need** | "Qual o principal processo que você quer automatizar?" | Dor clara e urgente |
| **Timeline** | "Para quando você precisa implementado?" | < 90 dias |

## Estrutura da Proposta (Directus)

```typescript
interface Proposal {
  id: string
  deal_id: string
  company_id: string
  status: 'draft' | 'sent' | 'approved' | 'signed' | 'rejected'
  total_value: number
  pdf_url?: string
  signed_at?: datetime
  items: ProposalItem[]          // serviços cotados
  milestones: ConsultingMilestone[] // entregas e valores
}
```
